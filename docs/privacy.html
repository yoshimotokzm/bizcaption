BizCaption プライバシーポリシー

最終更新日: 2026-05-04  |  提供者: Kazuma Yoshimoto

1. はじめに

Kazuma Yoshimoto（以下「当社」）は、リアルタイム音声文字起こし・翻訳アプリ「BizCaption」（以下「本アプリ」）を提供します。

本プライバシーポリシーは、本アプリにおけるデータの取得、利用、送信、保存、削除、ユーザーの選択について説明します。

2. 本アプリの基本方針

本アプリは、英語商談・会議等の理解を補助するため、ユーザーの音声を Microsoft Azure Speech に送信し、文字起こしと翻訳を行います。当社は、プライバシー保護と運用負荷の最小化のため、以下を基本方針とします。

  • 音声原本を本アプリ内に保存しません。
  • Azure Functions に音声本文、文字起こし本文、翻訳本文を保存しません。
  • 翻訳履歴はユーザーの端末内に保存します。
  • Azure Speech の audio logging / data logging を有効化しません。
  • 広告SDK、第三者広告、クロスアプリトラッキングを v1 では利用しません。

4. Microsoft Azure Speech への送信

本アプリは、ユーザーが明示的に同意した後、翻訳セッション中のマイク音声を Microsoft Azure Speech に送信します。送信目的は文字起こしと翻訳です。

利用リージョンは初期設定では Japan East を予定しています。将来リージョンを変更または追加する場合、本ポリシーを更新します。

当社は、Azure Speech の audio logging / data logging を有効化しません。リアルタイムの Speech to text / Speech Translation では、Microsoft は顧客が提供したデータを保持または保存しない前提で設計します。ただし、Microsoft Azure の処理は Microsoft の契約、Product Terms、DPA、SLA、プライバシー・セキュリティ条件に従います。

5. Azure Functions の役割

当社は、Azure Speech の秘密鍵をアプリ内に保存しないため、Azure Functions を利用して短命の Azure Speech token を発行します。Azure Functions は以下を行います。

  • StoreKit JWS の検証
  • DeviceCheck token の検証・中継
  • Azure Speech token の発行
  • Pro利用上限の最小監査
  • 不正利用・障害調査のための最小ログ保存

Azure Functions は、音声本文、文字起こし本文、翻訳本文、ユーザーメモ、シャドーイング進捗を保存しません。

6. 端末内に保存するデータ

本アプリは、翻訳履歴をユーザーの端末内に保存します。履歴には、会議・商談の内容、個人情報、営業秘密、顧客情報が含まれる場合があります。

ユーザーは、本アプリ内から履歴を削除できます。端末内の履歴は、ユーザーが削除するまで端末内に残ります。

当社は v1 では iCloud 同期を提供しません。将来 iCloud 同期等を提供する場合、本ポリシーを更新し、必要な同意と設定を追加します。

7. 利用目的

当社は、取得・処理するデータを以下の目的で利用します。

  • リアルタイム文字起こし・翻訳機能の提供
  • 翻訳履歴の表示、削除、復習機能の提供
  • 無料利用枠とPro利用上限の管理
  • サブスクリプションの認可、復元、状態確認
  • Azure Speech key の秘匿と短命token発行
  • 不正利用、過剰利用、障害、コスト急増の検知
  • ユーザーサポート
  • 法令、App Store Review、Apple / Microsoft契約への対応

10. ユーザーの選択と権利

ユーザーは、以下を行うことができます。

  • マイク権限をOS設定から拒否または取り消す
  • Azure cloud processing consent をアプリ設定から撤回する
  • 翻訳履歴を個別または一括削除する
  • サブスクリプションを App Store のアカウント設定から管理・解約する
  • 購入を復元する
  • 当社に問い合わせ、データに関する相談、削除依頼、訂正依頼を行う

Azure cloud processing consent を撤回した場合、新規翻訳セッションは開始できません。ただし、端末内の履歴閲覧、履歴削除、設定、サブスクリプション管理は引き続き利用できます。

11. App Store Privacy Nutrition Label との関係

App Store Connect では、実装内容に基づき、少なくとも以下を申告する予定です。

  • Audio Data: App Functionality、Trackingなし、原則ユーザーIDにリンクしない
  • User ID: StoreKit識別子等、App Functionality、Trackingなし
  • Purchase History: StoreKit購入履歴、App Functionality、Trackingなし

実装変更により収集データが変わる場合、App Store Privacy Nutrition Label と本ポリシーを更新します。

12. App Tracking Transparency

本アプリは v1 では広告目的のトラッキングを行いません。第三者広告SDK、データブローカー共有、クロスアプリ・クロスサイトのユーザー追跡を行いません。

そのため、v1 では App Tracking Transparency の許可ダイアログを表示しません。将来、トラッキングに該当する機能を導入する場合、事前に本ポリシーを更新し、必要な同意を取得します。

13. 子どものプライバシー

本アプリは、13歳未満の子どもを対象としていません。EEA等、より高い年齢基準が適用される地域では、その基準に従います。

当社が、保護者同意なく子どもの個人情報を取得したことを認識した場合、合理的な範囲で削除等の対応を行います。

14. 国際的なデータ移転

本アプリでは、Microsoft Azure の利用リージョンとして初期設定では Japan East を想定しています。ただし、Microsoft、Apple、ユーザーの所在地、サポート運用、法令上の要請により、データが他の国または地域で処理される場合があります。

当社は、適用される法令および Microsoft / Apple の契約条件に従い、必要な保護措置を講じます。

15. セキュリティ

当社は、以下のセキュリティ対策を講じます。

  • Azure Speech key をアプリ内に保存しない
  • Azure Functions で短命tokenを発行する
  • 通信時はHTTPS/TLSを利用する
  • Azure Key Vault で秘密情報を管理する
  • Functionsログに音声本文、文字起こし本文、翻訳本文、Azure token、JWS全文を保存しない
  • 端末内履歴にはiOS Data Protection等のOS保護を利用する

ただし、インターネット通信、端末、クラウドサービスに完全な安全性はありません。ユーザーは、端末ロック、OS更新、端末紛失時の対策を行ってください。

16. 変更

当社は、必要に応じて本ポリシーを変更することがあります。重要な変更を行う場合、本アプリ内、Webサイト、App Store の更新情報その他適切な方法で通知します。

17. 問い合わせ

本ポリシーまたはデータの取扱いに関する問い合わせは、サポートページよりご連絡ください。
